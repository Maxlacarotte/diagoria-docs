1. Les types de micros adaptés — lequel choisit ?

Micro de surface Boundary mic

Capsule plate posée sur une table. La conception boundary exploite la réflexion de la surface pour doubler la sensibilité dans l'hémisphère supérieur. Captation omnidirectionnelle très large, très discrète visuellement. Idéal pour poser sur le bureau ou coller sous le capot du PC.

Usage consultation : excellent — capte les deux interlocuteurs assis de part et d'autre du bureau. Exemple de référence pro : MXL AC-404 (introuvable facilement en FR, ~ 55 $ US).

Micro de conférence USB rond Reco principale

Capsule omnidirectionnelle 360° dans un boîtier cylindrique bas, posé au centre de la table. Très répandu, plug-and-play, souvent avec bouton muet LED. Portée typique 1–3 m. Format discret, cable USB 1,5–2 m.

Usage consultation : très bon rapport captation / prix / discrétion. C'est la catégorie recommandée pour le KIT.

Speakerphone de conférence Premium

Disque plus grand (15–20 cm), plusieurs capsules MEMS en cercle, traitement DSP embarqué (réduction de bruit, annulation d'écho). Capte une salle entière (jusqu'à 8–10 personnes). Généralement USB + Bluetooth. Souvent haut-parleur intégré — inutile pour DIAGORIA (la voix de l'IA sort par le PC).

Usage consultation : sur-dimensionné pour 2 personnes, mais qualité de captation supérieure si acoustique difficile (pièce très réverbérante).

Micro col-de-cygne USB Alternatif

Tige flexible orientable, capsule cardioïde ou omni en bout. Moins discret visuellement. Utile si le praticien veut orienter précisément le micro vers le patient. Plug-and-play possible selon modèle.

Usage consultation : moins adapté — la captation cardioïde rate le praticien si mal orienté. Préférer l'omnidirectionnel.

Pourquoi l'omnidirectionnel pour DIAGORIA ? En consultation, les deux voix viennent de directions opposées à 90–180° l'une de l'autre. Un micro cardioïde (directionnel) favorise une voix et rate l'autre. Un omnidirectionnel capture toute la sphère à égale sensibilité — les deux locuteurs sont captés sans réglage.

4. Conseil SETUP-MICRO (cohérence avec la doc existante)

Le fichier SETUP-MICRO.md indique que DIAGORIA utilise la Web Speech API de Chrome pour la transcription. Le micro USB apparaît dans Chrome comme périphérique d'entrée standard — il faut le sélectionner dans chrome://settings/content/microphone après branchement. Aucune configuration supplémentaire requise pour un micro USB plug-and-play.

  1. Brancher le micro USB

    Windows reconnaît automatiquement tout micro USB plug-and-play. Aucun pilote à installer pour TONOR TM20, eMeet ou Anker.

  2. Sélectionner dans Chrome

    Ouvrir chrome://settings/content/microphone → choisir le micro dans le menu déroulant (ex. : TONOR TM20 ou USB Audio Device).

  3. Vérifier dans DIAGORIA

    Lancer l'app → la barre de niveau doit bouger quand le patient parle. Si elle ne bouge pas : refaire l'étape 2.
