La carte des agents 🗺️

DIAGORIA n'est pas construit par une seule IA, mais par une équipe d'assistants spécialisés. Cette page montre, d'un coup d'œil, qui s'occupe de quoi — et comment ils évitent de se marcher dessus.

⚡ En 30 secondes

Sarabi est le chef d'orchestre : il ne code (presque) pas, il découpe le travail et le confie au bon spécialiste. Autour de lui, 8 agents ACTIFS font tourner le produit live (app, serveur, déploiement, sécurité, légal, data, design), et ~9 agents DORMANTS attendent leur phase (transcription, IA médicale, dossier patient…). Règle d'or : « un fichier = un seul propriétaire » — pour ne plus jamais crasher la prod comme le 03/06.

🧭 Comment lire ce schéma

Tout part de Sarabi. Quand tu lui dis « on bosse sur l'app », il route vers frontend ; « mets en ligne » → deployeur ; « est-ce légal ? » → referent-legal. Les agents ne se parlent pas directement entre eux : ils repassent toujours par Sarabi, ce qui évite les collisions.

💤 Les agents DORMANTS (phases futures)

Ils décrivent le produit cible (transcription live, IA médicale, dossier FHIR…). Ils ne tournent pas aujourd'hui : on en réveillera un à chaque nouvelle phase. Trois d'entre eux n'ont même pas encore de fiche écrite — ils existent seulement comme intention.

⚠️ Pourquoi cette règle « un fichier = un propriétaire » ?

Le 3 juin 2026, deux agents (backend et api-users) ont édité le même fichier main.py en même temps → un import manquant → le serveur en boucle de crash, tout le site en 404. Depuis : un seul orchestrateur (Sarabi), un seul propriétaire par fichier, et git status obligatoire avant chaque mise en ligne. La carte complète est dans OWNERSHIP.md.

✅ Ma reco concrète

Pour COMPRENDRE l'archi (qui fait quoi, qui possède quoi) → cette page. Garde-la en favori, elle ne périme pas.

Pour voir l'activité EN DIRECT → active Agent View (la fonctionnalité officielle d'Anthropic, mai 2026) : un seul tableau qui liste toutes tes sessions, leur état (en cours / en attente / terminé) et celles qui réclament une réponse. C'est exactement « qui travaille, là, maintenant ».

À savoir (limites honnêtes) : Agent View est en research preview, donc encore jeune ; le compteur d'agents dans la status line n'existe pas encore ; et les dashboards de la communauté sont plus riches mais demandent une installation technique. Pour ton usage débutant, le duo cette page + Agent View + la liste de tâches couvre 100 % du besoin.

💡 Astuce

« Agent View » s'appelle aussi la vue des agents / « manage agents ». Si /agents ne s'ouvre pas chez toi, c'est probablement une question de version de l'app ou de fonctionnalité à activer (research preview) : une mise à jour de Claude Code Desktop suffit en général à la débloquer.
